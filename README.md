# ShopFlow — E-commerce Backend API

ShopFlow is a backend system for an e-commerce platform built with Java and Spring Boot, focusing on scalable architecture, clean code principles, and enterprise-grade backend development practices.

The project simulates a real-world e-commerce system with emphasis on RESTful APIs, database modeling, and business logic implementation.

---

## 🚀 Tech Stack

- Java 21+
- Spring Boot
- Spring Web
- Spring Data JPA
- Hibernate
- PostgreSQL
- Maven
- Docker
- Git & GitHub

---

## 🧱 Architecture

The project follows a layered architecture approach:

- **Controller Layer** → Handles HTTP requests and responses
- **Service Layer** → Business logic implementation
- **Repository Layer** → Data access and persistence
- **Entity Layer** → Domain models mapped with JPA

This structure ensures separation of concerns, scalability, and maintainability.

---

## ⚙️ Features

- RESTful API development for e-commerce domain
- Product and order management
- Business logic implementation in service layer
- Relational database integration with Spring Data JPA
- Transaction management
- Data persistence with PostgreSQL
- Dockerized environment setup

---

## 🧠 Key Engineering Concepts

- Clean Code principles
- SOLID principles
- Layered architecture (Controller / Service / Repository)
- Object-Oriented Programming (OOP)
- REST API design
- Database normalization
- Transaction management and consistency

---

## 🗄️ Database Design

The system is built on relational modeling concepts:

- Entity relationships (One-to-Many, Many-to-Many)
- JPA annotations for ORM mapping
- PostgreSQL relational schema
- Data integrity with constraints and transactions
